**목적**: 한국 주식 단타·스윙 자동매매 봇. "스승님" 매매 철학의 코드화.

**철학**: "주가는 조작, 업황(섹터)과 수급만 본다." 펀더멘털(PER/PBR/ROE) 완전 배제. ① 섹터 쏠림(돈 몰리면 대장주 폭발) ② 수급(외/기/개) 만 봄. 폭발=+10%(VI), 추적 윈도우 D+20.

**핵심 전략(풀백 재폭발)**: NXT 폭등 → 본장 09:00~09:30 차익실현 하락 → 횡보(저점 유지) → 재폭발(+5~10%, 강하면 +20%)를 기계적 매매. → `core/pullback_detector.py`로 구현됨(09:20~09:40 윈도우, dry-run, 실매매 미연결).

**방향 전환(2026-04-27)**: 페이퍼 1일차 "신호 0건" — "같은 1분봉에 섹터 3종목 동시 통과" 가정이 비현실적(진짜 쏠림은 30분~수시간 누적). → 신호봇 → **데이터 누적봇**으로 전환. 임계치를 추측 말고 데이터로 도출.

**현재 상태(2026-06-27 갱신)**: 노트북 DB에 마이그레이션 m001~m009 적용 완료. `core/pipeline_runner.py`로 수집 파이프라인 **조립 완료** — 추적행 생성(ensure_tracking_rows, 빠졌던 핵심 연결고리) → 일봉 → 분봉 raw(NXT 장전 포함) → 3분봉 집계 → 돌파마킹 → 풀백 dry-run. `main_tracker.py`가 매일 16:00 `full_pipeline_job` 실행. 적립 기기는 [[data-accumulation-machine]](노트북). KIS 분봉은 당일만=백필 불가 → 평일 장중~16:00 노트북 켜둬야. **주의**: 마이그레이션 전 등록된 픽은 sector_pick_events가 없어 추적 안 됨(재등록 필요).

**자매 프로젝트**: ai-moneyingbot(스승님 게시물 4.2만 RAG) 데이터와 합쳐 신호 룰 재설계 예정.
